# Kortix V0

Standalone repo-native Kortix proof of concept.

```bash
pnpm --filter @kortix/v0 dev
```

Open `http://127.0.0.1:4310`.

## What It Does

- Creates/imports a Kortix Project whose source of truth is a git repo.
- Reads any repo the machine can access and detects whether it is already Kortix-shaped.
- Parses `kortix.toml` as the project contract, including declared required/optional env vars.
- Reads project files from a local bare Git mirror instead of from a live sandbox.
- Stores project vault values encrypted in the local control-plane DB and injects them into sandboxes via provider env vars.
- Creates session runs by making a unique branch named `kortix/session/<session_id>`.
- Provisions a Daytona sandbox from `DAYTONA_SNAPSHOT` or `kortix-sandbox-vdev-<gitsha>-poc`, clones and pulls the session branch into `/workspace`, starts OpenCode on port `4096`, and optionally dispatches the initial prompt.

## Project Manifest

Every Kortix project repo can include `kortix.toml`.

Schema:

- canonical id: `https://schemas.kortix.com/project/v0`
- local dev route: `http://127.0.0.1:4310/schemas/project/v0`
- repo copy: `apps/kortix-v0/schemas/project-v0.schema.json`

Minimal example:

```toml
schema = "https://schemas.kortix.com/project/v0"
name = "Acme"

[runtime]
engine = "opencode"
mode = "ephemeral"

[opencode]
config_dir = ".opencode"
config = ".opencode/opencode.jsonc"

[env]
required = []
optional = []
```

The default starter is intentionally just OpenCode:

```txt
kortix.toml
.opencode/opencode.jsonc
.gitignore
README.md
```

No Kortix agent, skills, provider config, context, memory, or state is generated by default. Add those later only when the project needs them.

## CLI

```bash
apps/kortix-v0/bin/kortix init acme --dir ./acme
apps/kortix-v0/bin/kortix project:create --managed --name Acme # creates a private GitHub repo for now

pnpm --filter @kortix/v0 cli init acme --dir ./acme
pnpm --filter @kortix/v0 cli project:create --repo-url https://github.com/acme/acme-kortix.git
pnpm --filter @kortix/v0 cli project:create --repo-url git@github.com:acme/acme-kortix.git --name Acme
pnpm --filter @kortix/v0 cli project:create --managed --name Acme # creates a private GitHub repo for now
pnpm --filter @kortix/v0 cli project:inspect --repo-url https://github.com/acme/acme-kortix.git
pnpm --filter @kortix/v0 cli projects:list
pnpm --filter @kortix/v0 cli secrets:list --project <project-id>
pnpm --filter @kortix/v0 cli secrets:set --project <project-id> --key ANTHROPIC_API_KEY --value sk-...
pnpm --filter @kortix/v0 cli session:create --project <project-id> --prompt "Inspect the project."
```

## Runtime Daemon

The sandbox runtime starts through one tiny installable entrypoint:

```bash
apps/kortix-v0/runtime/install-kortix-daemon.sh /usr/local/bin/kortix-daemon
kortix-daemon start
```

For now it only resolves the project `.opencode` directory and starts `opencode serve`.

## Env

Copy `.env.example` to `.env` or rely on the root/app env files. The POC reads, in order:

- repo root `.env`
- `apps/api/.env`
- `apps/kortix-v0/.env`

It stores temporary API-server metadata, the local SQLite DB, Git mirrors, and encrypted test vault values under `apps/kortix-v0/.local/` by default. Override that with `KORTIX_V0_DATA_DIR` when needed.
