title: Triggers
description: A trigger starts a session automatically, on a schedule or from a webhook.
---

A **trigger** spawns a [session](/docs/concepts/sessions) with no person present. Use a trigger to automate recurring or event-driven work.

## Trigger types

Kortix supports two trigger types:

- **cron** — runs on a schedule you set.
- **webhook** — runs when an external service sends a signed request to the project's webhook URL.

## Where triggers live

You define triggers in the project manifest, `kortix.yaml`. Each trigger holds a prompt that starts the fired session's first message.

Kortix stores runtime state — such as the last fire time and its status — separately from the manifest. Firing a trigger does not create a commit.

## Session strategy

A fired trigger can start a new session, resume an earlier one, or target a session you pin. See the [triggers reference](/docs/reference/triggers) for the full list of session strategies and the manifest fields.
