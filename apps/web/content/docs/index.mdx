---
title: Kortix
description: Run agents in disposable VM sandboxes against your own GitHub repos.
---

Kortix is a control plane for agentic work. Each project is **one GitHub
repo**. Each conversation runs in **one isolated VM sandbox** with that
repo cloned onto its own ephemeral branch — the agent (OpenCode under
the hood) can install, edit, run anything; only what it commits and
pushes survives. The user reviews the branch and merges when ready.

The contract between you and the platform is a single file at your
repo root: **`kortix.toml`**. Everything else — agent personas, skills,
slash commands, the Dockerfile for your sandbox — lives inside a
`.kortix/` folder next to it. That repo *is* the project. There's no
external dashboard state, no separate "project settings" UI to keep in
sync. Edit a file, commit, the next session picks it up.

Triggers (cron + webhook), env-var spec, the sandbox image, and
deployable apps all live in `kortix.toml` as TOML blocks. The dashboard
just reads + writes that file via the GitHub API. You can edit it from
inside a session, from the web UI, or from your own editor —
round-trips cleanly either way.

## Quickstart

```sh
# 1. Scaffold a new project locally
bunx @kortix/cli init my-project
cd my-project

# 2. Push to GitHub
git remote add origin https://github.com/you/my-project.git
git push -u origin main

# 3. Register the repo as a Kortix project (via the dashboard)
#    and start your first session.
```

Or skip the local step entirely and use **Create new project** in the
dashboard — it creates the GitHub repo, scaffolds the starter, registers
the project, and opens the first session in one flow.

## The shape of a Kortix repo

```
my-project/
├── kortix.toml                              ← project manifest (Kortix reads this)
├── README.md
└── .kortix/                                 ← everything Kortix-owned
    ├── Dockerfile                           ← sandbox base image
    └── opencode/                            ← opencode's config dir
        ├── opencode.jsonc
        ├── agents/kortix.md
        ├── skills/kortix-system/SKILL.md
        └── tools/show.ts
```

Two paths in the manifest control where these live. Defaults are
`.kortix/Dockerfile` and `.kortix/opencode` — relocate either if you
need to share an opencode config across multiple repos.
