For contributors running this repository locally. Requires Docker, Node 22, and pnpm 8.

```sh
pnpm install
pnpm dev          # web + API + Supabase + tunnel
```

## How secrets work

The API's secrets live **encrypted in the repo** with [dotenvx](https://dotenvx.com). Every value in `apps/api/.env` is AES-encrypted (`KEY=encrypted:…`). The **public key** that encrypts them sits in the file and is safe to commit; the **private key** that decrypts them never touches git. So the secrets are safe in version control — only the key is sensitive.

There are three profiles, one file each, every one with its own keypair:

| Profile | File | Used by |
| --- | --- | --- |
| local | `apps/api/.env` | `pnpm dev` |
| dev | `apps/api/.env.dev` | `dotenvx run -f apps/api/.env.dev -- …` |
| prod | `apps/api/.env.prod` | `dotenvx run -f apps/api/.env.prod -- …` (run locally against prod when needed) |

The private keys are held off your machine in [Dotenv Armor](https://dotenvx.com/armor). `pnpm dev` decrypts them **in memory** at boot via `dotenvx run` — nothing plaintext is ever written to disk. (The deployed prod runtime still gets its own env injected; `.env.prod` here is only for running locally against prod.)

## First time on a new machine

<Steps>
<Step>
### Install the key manager and log in
```sh
curl -sfS https://dotenvx.sh/armor | sh
dotenvx-armor login
```
</Step>
<Step>
### Pull the keys from the cloud
```sh
cd apps/api
for f in .env .env.dev .env.prod; do dotenvx-armor pull -f "$f"; done
```
</Step>
<Step>
### Enable the commit hooks + run
```sh
git config core.hooksPath .githooks   # auto-encrypts secrets on commit
pnpm dev
```
</Step>
</Steps>

The pre-commit hook **auto-encrypts** `apps/api/.env` / `.env.dev` (so a hand-typed plaintext value is sealed before it can be committed) and blocks the commit if any unencrypted `.env` slips through.

## Everyday

| Task | Command |
| --- | --- |
| Run locally | `pnpm dev` |
| Read a secret | `dotenvx get KEY -f apps/api/.env` |
| Add / change a secret | `dotenvx set KEY value -f apps/api/.env` (or `.env.dev`), then commit |
| Share a new/rotated key with the team | `dotenvx-armor push -f <file>` (one file at a time) |

<Callout type="warn">
Never write plaintext into `apps/api/.env` — it's encrypted and committed. Machine-local overrides go in the gitignored `apps/api/.env.local`, which Bun loads at higher precedence.
</Callout>
