This is the loop you'll use every day, in the dashboard.

<Steps>

<Step>
### Create a project

A [project](/docs/concepts/projects) is one company, product, or idea. From your
projects list, create one — start with a Kortix-managed repo, or import an
existing GitHub repo (for developers).
</Step>

<Step>
### Run a session

Open the project and start a [session](/docs/concepts/sessions). Describe the
task like you would for a teammate. The agent works in its own private, disposable
copy of the project — you can watch and reply as it goes.
</Step>

<Step>
### Review and keep it

When the agent finishes, it opens a
[change request](/docs/concepts/change-requests) — a summary and the exact
changes. Review it and **merge** to keep the work, or ask for changes. Nothing
touches your project until you merge.
</Step>

</Steps>

That's it. From here: give the agent your [secrets](/docs/guides/managing-secrets)
and [tools](/docs/guides/connecting-tools), [automate](/docs/guides/automating-work)
recurring work, or invite your [team](/docs/concepts/accounts).
